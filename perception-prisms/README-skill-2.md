# Skill 2: Призмы восприятия — инструкция для запуска

## Что это и зачем

Этот инструмент берёт таблицу с размеченными сигналами из отзывов (результат Skill 1) и отвечает на главный вопрос маркетинга: **кто читает эти отзывы и как они их воспринимают?**

Один и тот же отзыв читают по-разному:
- Человек, который ищет подарок, обращает внимание на упаковку и «премиальность»
- Человек с сухой кожей смотрит на увлажнение и состав
- Экономный покупатель ищет долговечность и цену за объём

Skill 2 строит «призмы» — фильтры восприятия — и для каждой говорит: **вот что писать в карточке, вот какие отзывы показывать, вот что никогда не обещать**.

**Аналогия:** если Skill 1 — это аудиозапись разговора покупателей, то Skill 2 — это переводчик, который объясняет, что люди на самом деле имели в виду, и советует, как говорить с каждым из них.

## Что нужно подготовить

### Обязательно

**1. Файл `signal-markup.md`** — результат работы Skill 1.
Это таблица с отзывами и размеченными сигналами, которую создал Skill 1. Она уже у тебя есть после первого прогона.

**2. Файл `card-review.md`** — твой голосовой обзор карточки товара.
Это краткое описание того, что написано в карточке на маркетплейсе: какие обещания, какие образы, на что делается акцент.

**Как создать `card-review.md`:**
Посмотри на карточку товара и запиши (или продиктуй голосом, потом перепиши) ответы на вопросы:
- Что главное обещание продукта?
- Какие свойства подчёркиваются?
- Какие образы используются (фото, описание)?
- Для кого позиционируется?
- Что написано в описании и буллитах?

Пример готового файла: `examples/card-review-example.md`

### Опционально

**3. Файл `prisms-library.md`** — призмы от предыдущего товара.
Нужен только если ты уже запускал этот skill раньше и хочешь проверить, работают ли те же призмы для нового товара. После каждого прогона Skill 2 сам создаёт файл для переноса — скопируй его.

Пример: `examples/prisms-library-example.md`

**4. Файл `card-reviews-competitors.md`** — обзоры карточек конкурентов.
Создаётся так же, как `card-review.md`, только для нескольких конкурентов. Нужен, если хочешь найти незанятые позиции в конкурентном поле.

## Пошаговая инструкция запуска

### Шаг 1: Открой нужный LLM

Подойдёт ChatGPT (GPT-4 или выше), Claude (Sonnet или Opus) или другая мощная языковая модель.

### Шаг 2: Скопируй промпт

Открой файл `skill-2-perception-prisms.prompt.md` и скопируй **весь** его текст.

### Шаг 3: Вставь промпт

Вставь скопированный текст в чат как первое сообщение (или как системный промпт, если интерфейс это поддерживает).

### Шаг 4: Приложи файлы

Прикрепи к чату:
- `signal-markup.md` (обязательно)
- `card-review.md` (обязательно)
- `prisms-library.md` (если есть)
- `card-reviews-competitors.md` (если есть)

### Шаг 5: Запусти

Напиши в чат:

```
Запусти Skill 2
```

Если хочешь прогон для конкурента, напиши:

```
Запусти Skill 2 для конкурента [название]
```

### Шаг 6: Дожди результата

Прогон займёт несколько минут. Модель выдаст полный результат по всем модулям. Если она остановилась на середине — напиши «продолжай».

### Шаг 7: Сохрани результат

Скопируй весь результат в файл `perception-prisms-[дата].md`.

Отдельно скопируй последний блок **«Файл призм для переноса»** — сохрани его как `prisms-library.md` для следующего прогона.

## Как читать результат

### Структура результата (по порядку)

**ПРЕДШАГ: Разрывы карточка ↔ отзывы**
Таблица с разрывами между тем, что обещает карточка, и тем, что реально пишут покупатели. Это список задач для редактирования карточки.

**Призмы восприятия**
Каждая призма — это один тип мотивации. Читай:
- **●** = сильная призма, подтверждена отзывами → обязательно учитывать
- **◐** = есть признаки, но данных мало → стоит учитывать
- **○** = гипотеза → можно проверить

Смотри на % — он показывает, какая доля покупателей читает через эту призму.

**Углы подачи**
Для каждой ● и ◐ призмы: конкретный оффер, обещания, запреты, продающие отзывы. Это готовый материал для переработки карточки.

**Эвристики и кросс-категорийные офферы**
Скрытые связи в отзывах и идеи из других категорий. Используй как источник нестандартных гипотез.

**Сводная таблица**
Одностраничный обзор всех призм с приоритетами. Начинай отсюда, если нет времени читать всё.

### Что делать дальше

1. Посмотри сводную таблицу — определи 1–2 приоритетные призмы
2. Возьми углы подачи для этих призм
3. Перепиши карточку, используя рекомендации из «Что изменить в карточке»
4. Проверь раздел «Что нельзя обещать»
5. Скопируй «Файл призм для переноса» в `prisms-library.md`

## Как запускать для конкурентов

### Порядок прогонов

**Первый прогон (собственный товар):**
1. Запусти Skill 2 с `signal-markup.md` + `card-review.md`
2. Сохрани результат
3. Скопируй «Файл призм для переноса» → `prisms-library.md`

**Второй прогон (конкурент 1):**
1. Создай `signal-markup-competitor-1.md` через Skill 1 по отзывам конкурента
2. Создай `card-review-competitor-1.md` — обзор карточки конкурента
3. Запусти Skill 2 с этими файлами + `prisms-library.md` (из первого прогона)
4. Смотри раздел «Незанятые призмы конкурентного поля»

**Третий и следующие прогоны:**
Добавляй в `card-reviews-competitors.md` обзоры всех конкурентов, передавай в один прогон.

### Что даёт прогон по конкурентам

- Какие призмы занял конкурент (его позиционирование)
- Какие призмы никто не занял (возможность)
- Где конкурент врёт (его разрывы карточка ↔ отзывы — твоё преимущество)

## Как корректировать результат

Если результат не устраивает — напиши в тот же чат одну из этих фраз:

**Если призма кажется неверной:**
```
Призма [название] кажется слишком размытой. Уточни, что именно отличает эту мотивацию от [другой призмы].
```

**Если хочешь углубить один угол:**
```
Развернуть угол подачи для призмы [название]: нужно больше конкретики в запретных словах и продающих отзывах.
```

**Если не хватает примеров из отзывов:**
```
По призме [название] нужны цитаты из отзывов. Приведи конкретные ID и фрагменты.
```

**Если хочешь проверить гипотезу:**
```
Проверь гипотезу: в данных есть призма [твоё предположение]? Найди якорные отзывы или объясни, почему её нет.
```

**Если хочешь переделать угол подачи:**
```
Переделай оффер для призмы [название]: он должен быть более конкретным и не использовать слово [слово].
```

**Если хочешь добавить новую призму вручную:**
```
Добавь призму [название] с мотивацией [описание]. Найди подходящие отзывы и оцени уверенность.
```

## Частые вопросы

**Сколько призм обычно получается?**
3–7. Меньше трёх — данных мало или отзывы очень однородны. Больше семи — скорее всего, есть лишние (похожие мотивации объединяются).

**Что делать, если все призмы с уверенностью ○?**
Значит, отзывы не раскрывают мотивации — люди писали только о результате, не о том, зачем покупали. В этом случае используй внешние эвристики категории (Блок B) и увеличь долю гипотез. Результат всё равно полезен, но помечен как «требует проверки».

**Почему нет числовых оценок (1–10, веса)?**
Потому что «8 из 10» ничего не значит без контекста. Для человека, выбирающего подарок, упаковка важна на 10, а для экономного покупателя — на 2. Числа создают иллюзию точности там, где её нет.

**Как часто перезапускать Skill 2?**
При обновлении отзывов раз в квартал, или когда накопилось 50+ новых отзывов, или перед крупным изменением карточки.
