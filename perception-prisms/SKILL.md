# Skill: perception-prisms

## Назначение

Skill `perception-prisms` берёт реестр сигналов из Skill 1 и пропускает его через «призмы восприятия» — модели того, как разные мотивации меняют интерпретацию одних и тех же отзывов.

Это не аналитический отчёт по частотам сигналов.

Это инструмент для ответа на вопрос: **что писать в карточке, и для кого?**

## Когда использовать

- есть `signal-markup.md` — результат Skill 1;
- есть `card-review.md` — обзор карточки товара;
- нужно понять, как разные покупатели читают одни и те же отзывы;
- нужны конкретные офферы, обещания и запреты для карточки.

Не использовать этот skill для:

- первичной разметки сигналов (это Skill 1);
- сравнения нескольких товаров без призматической логики;
- автоматического редактирования карточки.

## Главная архитектурная идея

Сигнал не имеет объективного веса. Вес существует только внутри призмы.

Одно свойство может быть КРИТИЧНО для покупателя-дарителя и НЕВАЖНО для экономного рационалиста. Поэтому сначала строятся призмы, потом внутри каждой оцениваются сигналы.

## Вход

**Обязательные:**
1. `signal-markup.md` — результат Skill 1
2. `card-review.md` — голосовой обзор карточки товара

**Опциональные:**
3. `prisms-library.md` — призмы от предыдущего товара (режим «с багажом»)
4. `card-reviews-competitors.md` — обзоры карточек конкурентов

Пример формата входов: [examples/card-review-example.md](examples/card-review-example.md), [examples/prisms-library-example.md](examples/prisms-library-example.md)

## Выход

Skill должен вернуть:

1. **ПРЕДШАГ**: таблица разрывов «Карточка ↔ Отзывы»
2. **Призмы**: 3–7 призм восприятия с оценкой сигналов внутри каждой
3. **Углы подачи**: оффер, обещания, запреты, продающие отзывы — для каждой ● и ◐ призмы
4. **Эвристики**: цепочки из отзывов, внешние стереотипы, скептические интерпретации, кросс-категорийные офферы
5. **Финальные секции**: сводная таблица, молчаливое большинство, файл для переноса

## Pipeline flow

```text
Input: signal-markup.md + card-review.md [+ опционально prisms-library.md]
    ↓
[ПРЕДШАГ] → таблица разрывов «Карточка ↔ Отзывы»
    ↓
[МОДУЛЬ 1] → призмы восприятия (якорные отзывы → кластеризация → проверка пропусков)
    ↓
[МОДУЛЬ 2] → углы подачи для каждой ● и ◐ призмы
    ↓
[МОДУЛЬ 3] → эвристики + скепсис + кросс-категорийные офферы
    ↓
[ФИНАЛ] → сводная таблица + файл призм для переноса (prisms-library.md)
```

## Два режима

**«С нуля»** (нет `prisms-library.md`):
- Призмы строятся из якорных отзывов
- Уверенность определяется по количеству якорей

**«С багажом»** (есть `prisms-library.md`):
- Призмы из библиотеки проверяются на данных текущего товара
- Результат: подтверждена ✅ / ослаблена ⚠️ / не найдена ❌
- Новые призмы добавляются к существующим

## Ключевые принципы

1. **Призма ≠ личность.** Это фильтр восприятия, не демографический портрет.
2. **Нет объективного веса.** Только КРИТИЧНО / ОПАСНО / НЕВАЖНО внутри призмы.
3. **Скептицизм к негативу.** Может ли покупатель объяснить плохой отзыв ошибкой автора?
4. **Молчаливое большинство.** Обязательная призма в каждом прогоне.
5. **Главный выход — углы подачи.** Призмы — промежуточные расчёты.

## Жёсткие запреты

1. Не создавать демографические портреты («женщина 35 лет»).
2. Не присваивать числовые веса сигналам.
3. Не пропускать призму «Молчаливое большинство».
4. Не делать «объективный» рейтинг товара.
5. Не придумывать сигналы — только из `signal-markup.md`.
