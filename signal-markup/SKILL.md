# Skill: signal-markup

## ⛔ ГЛАВНОЕ ПРАВИЛО

**Это задание по написанию текста, НЕ по программированию.**

Ты читаешь отзывы и пишешь один Markdown-файл. Не пишешь код. Не создаёшь JSON. Не запускаешь команды.

## Что делать

1. Прочитать файл с отзывами
2. Написать `master_registry.md` с 5 блоками (A–E)
3. Всё. Один файл. Готово.

## Полная инструкция

Все правила, формат выхода и примеры — в одном файле:

→ **[prompts/extract-signals.md](prompts/extract-signals.md)**

Не нужно читать другие файлы. Всё собрано в промпте.

## Быстрая справка

| Emoji | Категория |
|-------|-----------|
| 🟢 | позитив — конкретный плюс |
| 🔴 | негатив — конкретный минус |
| ⚪ | ожидание — что хотел получить |
| 🔵 | сомнение — риск до покупки |
| 🟣 | сравнение — «лучше/хуже чем X» |
| 🟠 | сценарий — зачем/для кого |

## Запрещено

- ❌ Скрипты, парсеры, код (Python, JS, bash)
- ❌ JSON, CSV, любые форматы кроме .md
- ❌ Команды в терминале (grep, wc, python)
- ❌ Промежуточные файлы
- ❌ Аналитика, группировка, отчёты
