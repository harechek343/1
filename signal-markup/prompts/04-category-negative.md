# Промпт 04: Негативные сигналы

**Шаг pipeline:** 04  
**Вход:** `primary_signals[]` (только категория «негатив») + исходная таблица + `B_total` + `B_clean`  
**Выход:** `category_report[негатив]`

---

## Задача

Развернуть негативные сигналы в полные цитаты, сгруппировать в темы, выделить критичные сигналы (вред, брак, безопасность), подсчитать частоту.

---

## Шаг 1: Развёртка по unit_index

Для каждого сигнала из primary_signals[негатив]:
1. Взять unit_index (#)
2. Найти соответствующую строку в исходной таблице
3. Привести **полную цитату** из оригинального текста
4. Выделить **жирным** конкретный сигнал внутри цитаты

Формат строки:
```
#unit_index | имя | «полная цитата с **выделенным сигналом**»
```

Пример:
```
#3 | Покупатель | «**Флакон пришёл треснутый**, половина вытекла. Очень расстроена.»
#6 | Света | «**Думала, что запах мягче, а он резкий.** Не то, чего ожидала.» [has_expectation]
```

Если у сигнала был вторичный тег `has_expectation: true` — пометить `[has_expectation]`.

---

## Шаг 2: Группировка в темы

Сгруппировать по предмету негатива. Название темы — **2-4 слова**, почти-цитатная формулировка.

Примеры:
- «флакон пришёл треснутый» ✓
- «упаковка» ✗ (слишком абстрактно)
- «запах слишком резкий» ✓
- «запах не нравится» ✗ (неточно)

Для каждой темы:
- Перечислить все unit_index с цитатами
- Подсчитать R_signal
- Вычислить R_signal/B_total (%) и R_signal/B_clean (%)

Сортировать по убыванию R_signal.

---

## Шаг 3: Блок критичных сигналов

Выделить отдельным блоком сигналы по критичным темам:
- **Вред здоровью**: жжение, ожог, раздражение кожи, аллергическая реакция
- **Брак**: треснул, сломан, не открывается, пришёл разбитый
- **Подмена**: состав другой, другой товар, поддельный
- **Безопасность**: опасно, противопоказано, предупреждение не указано
- **Протечки**: вытекло, протекло, залило упаковку

Для критичных сигналов — выделить даже если R=1.

Формат:
```
⚠️ КРИТИЧНЫЕ СИГНАЛЫ:
🔴 Вред/аллергия: N упоминаний
   #23 | Марина | «после первого применения **сильное жжение**, красные пятна»
🔴 Брак упаковки: N упоминаний
   #3 | Покупатель | «флакон пришёл треснутый, половина вытекла»
```

---

## Шаг 4: Подсчёт

```
R_signal = количество уникальных unit_index в теме
% от B_total = R_signal / B_total × 100
% от B_clean = R_signal / B_clean × 100
```

---

## Шаг 5: Динамика по времени

Если есть поле «Дата»:
- Распределить темы по периодам
- Отметить усиление / ослабление / новые темы
- Если нет дат → `NO DATA`

---

---

## Жёсткие запреты

- Не занижать критичные сигналы, даже при R=1
- Не объединять «запах резкий» и «запах слабый» в одну тему «проблемы с запахом»
- Не интерпретировать причины — только фиксировать что написано
- Не пропускать сигналы с `has_expectation: true` — они уже правильно размечены как негатив
