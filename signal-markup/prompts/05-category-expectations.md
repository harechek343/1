# Промпт 05: Ожидания

**Шаг pipeline:** 05  
**Вход:** `primary_signals[]` (только категория «ожидание») + исходная таблица + `B_total` + `B_clean`  
**Выход:** `category_report[ожидание]`

---

## Задача

Развернуть сигналы ожиданий в полные цитаты, типизировать, сформулировать почти-цитатно, подсчитать частоту.

**Ожидание** — это не постфактум. Это то, что покупатель хотел получить, считал обещанным, по какому признаку судил о товаре.

---

## Важное разграничение

Ожидания **не смешивать** с сомнениями:
- **Ожидание** = «что хотел получить» (направлено на желаемый результат)
- **Сомнение** = «чего боялся» (направлено на риск)

Эти категории разведены в отдельные промпты. Сомнения — в промпте 06.

---

## Шаг 1: Развёртка по unit_index

Для каждого сигнала из primary_signals[ожидание]:
1. Взять unit_index (#)
2. Найти исходный текст
3. Привести **полную цитату**
4. Выделить **жирным** формулировку ожидания

Формат:
```
#unit_index | имя | «полная цитата с **выделенным ожиданием**»
```

---

## Шаг 2: Типизация

Для каждого сигнала определить тип:

| Тип | Описание | Маркеры |
|-----|---------|---------|
| `[что хотел получить]` | Конкретный желаемый результат | «хотела», «ожидала», «рассчитывала» |
| `[важный критерий]` | Свойство, критичное при выборе | «брала именно за», «важно было», «выбирала по» |
| `[как должно работать]` | Ожидаемый механизм действия | «должно было», «по описанию», «заявлено» |
| `[как должно ощущаться]` | Сенсорные ожидания (запах, текстура) | «думала, что запах», «ожидала текстуру» |
| `[что считал обещанным]` | На основе описания/упаковки | «написано», «на сайте», «по составу» |

---

## Шаг 3: Группировка в темы

Группировать по предмету ожидания. Название темы — **почти-цитатная формулировка**, 3-7 слов.

Примеры:
- «ожидала более мягкий запах» ✓
- «запах» ✗ (слишком абстрактно)
- «хотела быстрый видимый результат» ✓
- «результат» ✗

Для каждой темы:
- Перечислить сигналы с цитатами и типами
- Подсчитать R_signal
- Вычислить проценты от B_total и B_clean

---

## Шаг 4: Подсчёт

```
R_signal = количество уникальных unit_index в теме
% от B_total = R_signal / B_total × 100
% от B_clean = R_signal / B_clean × 100
```

---

## Шаг 5: Динамика по времени

Если есть даты → распределить по периодам, отметить изменения.  
Если нет → `NO DATA`

---

## Жёсткие запреты

- **Не включать сомнения** — их промпт отдельный (06)
- **Не включать чистый постфактум** без формулировки ожидания (это негатив, уже размечен)
- **Не сокращать цитаты** — приводить полные
- **Не объединять ожидание+позитив** и **ожидание+негатив** — они уже разобраны по составному протоколу
- **Не придумывать ожидания** из подтекста — только то, что явно написано
