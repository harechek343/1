# Промпт 06: Сомнения

**Шаг pipeline:** 06  
**Вход:** `primary_signals[]` (только категория «сомнение») + исходная таблица + `B_total` + `B_clean`  
**Выход:** `category_report[сомнение]`

---

## Задача

Развернуть сигналы сомнений в полные цитаты, типизировать, сформулировать почти-цитатно, подсчитать частоту.

**Сомнение** — это риск или неопределённость **до покупки**. Не постфактум.

---

## Важное разграничение

Сомнения **не смешивать** с ожиданиями:
- **Сомнение** = «чего боялся / что было неясно» (неопределённость, риск)
- **Ожидание** = «что хотел получить» (желаемый результат)

**Жёсткое правило:** если это уже случившийся факт — **НЕ сомнение**, а негатив.

Примеры разграничения:
- «Боюсь, что запах будет слишком сильным» → сомнение ✓ (риск до покупки)
- «Запах оказался слишком сильным» → негатив ✗ (уже случился)
- «Не уверена, подойдёт ли» → сомнение ✓
- «Не подошёл» → негатив ✗

---

## Шаг 1: Развёртка по unit_index

Для каждого сигнала из primary_signals[сомнение]:
1. Взять unit_index (#)
2. Найти исходный текст
3. Привести **полную цитату**
4. Выделить **жирным** формулировку сомнения

Формат:
```
#unit_index | имя | «полная цитата с **выделенным сомнением**»
```

---

## Шаг 2: Типизация

| Тип | Описание | Маркеры |
|-----|---------|---------|
| `[риск до покупки]` | Конкретное опасение | «боюсь», «опасаюсь», «переживаю» |
| `[неопределённость характеристики]` | Непонятное свойство | «не уверена», «неясно», «непонятно» |
| `[страх ошибки выбора]` | Боюсь купить не то | «а вдруг не то», «не знаю, правильно ли» |
| `[вопрос без ответа]` | Прямой вопрос | «подойдёт ли?», «как понять?» |

---

## Шаг 3: Группировка в темы

Группировать по предмету сомнения. Название — **почти-цитатная формулировка**, 3-7 слов.

Примеры:
- «подойдёт ли для жирной кожи» ✓
- «тип кожи» ✗ (абстрактно)
- «боюсь аллергической реакции» ✓
- «аллергия» ✗

Для каждой темы:
- Перечислить сигналы с цитатами и типами
- Подсчитать R_signal
- Вычислить R_signal/B_total (%) и R_signal/B_clean (%)

---

## Шаг 4: Подсчёт

```
R_signal = количество уникальных unit_index в теме
% от B_total = R_signal / B_total × 100
% от B_clean = R_signal / B_clean × 100
```

## Жёсткие запреты

- **Не включать ожидания** — их промпт отдельный (05)
- **Не включать уже случившиеся факты** — они размечены как негатив на шаге 02
- **Не придумывать сомнения** из подтекста
- **Не объединять разные риски** в одну абстрактную тему
