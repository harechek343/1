# Промпт 08: Сценарии

**Шаг pipeline:** 08  
**Вход:** `primary_signals[]` (только категория «сценарий») + исходная таблица + `B_total` + `B_clean`  
**Выход:** `category_report[сценарий]`

---

## Задача

Развернуть сигналы сценариев, типизировать, сгруппировать в темы, выделить критичные (сломан главный сценарий), подсчитать частоту.

**Сценарий** — это контекст использования: зачем купили, как используют, для кого, где, когда, какую задачу решают.

---

## Шаг 1: Развёртка по unit_index

Для каждого сигнала из primary_signals[сценарий]:
1. Взять unit_index (#)
2. Найти исходный текст
3. Привести **полную цитату**
4. Выделить **жирным** формулировку сценария

Формат:
```
#unit_index | имя | «полная цитата с **выделенным сценарием**»
```

---

## Шаг 2: Типизация

| Тип | Описание | Маркеры |
|-----|---------|---------|
| `[зачем купил]` | Цель покупки | «брала для», «покупала чтобы» |
| `[как использует]` | Способ применения | «наношу каждое утро», «применяю так-то» |
| `[для кого]` | Получатель / пользователь | «дочке», «маме», «себе», «мужу» |
| `[контекст]` | Условия применения | «на работу», «в путешествие», «летом» |
| `[решаемая задача]` | Проблема, которую решает | «помогает с», «справляется с», «для сухой кожи» |

---

## Шаг 3: Группировка в темы

Группировать по типу сценария. Название темы — **почти-цитатная формулировка**, 3-6 слов.

Примеры:
- «подарок маме / близким» ✓
- «подарок» ✗ (абстрактно)
- «ежедневный уход за руками» ✓
- «уход» ✗

Для каждой темы:
- Перечислить сигналы с цитатами и типами
- Подсчитать R_signal
- Вычислить проценты

---

## Шаг 4: Критичные сценарии

Выделить отдельным блоком:

**«Сломан главный сценарий»** — товар не справляется со своим основным назначением:
- Крем для рук, которым нельзя пользоваться руками
- Шампунь, от которого выпадают волосы
- Увлажняющий крем, который сушит

Формат:
```
🔴 СЛОМАН ГЛАВНЫЙ СЦЕНАРИЙ: N упоминаний
   #67 | Покупатель | «крем для чувствительной кожи вызвал раздражение у всей семьи»
```

**«Не тот формат»** — товар не подходит для заявленного использования:
```
⚠️ НЕ ТОТ ФОРМАТ: N упоминаний
```

---

## Шаг 5: Подсчёт

```
R_signal = количество уникальных unit_index в теме
% от B_total = R_signal / B_total × 100
% от B_clean = R_signal / B_clean × 100
```

---

## Шаг 6: Динамика по времени

Если есть даты → отметить изменения в паттернах использования.  
Если нет → `NO DATA`

---

## Жёсткие запреты

- **Не включать ожидания под видом сценариев** — «хотела, чтобы помогло с сухостью» — это ожидание, не сценарий
- **Не придумывать сценарии** из подтекста — только явно описанный контекст
- **Не объединять разные задачи** в одну абстрактную тему «уход»
