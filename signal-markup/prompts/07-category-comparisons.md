# Промпт 07: Сравнения

**Шаг pipeline:** 07  
**Вход:** `primary_signals[]` (только категория «сравнение») + исходная таблица + `B_total` + `B_clean`  
**Выход:** `category_report[сравнение]`

---

## Задача

Развернуть сигналы сравнений, выделить объект сравнения X, нормализовать X, сгруппировать, выявить критичные сравнения (подмены).

---

## Требование к сравнению

**Сравнение требует явного X.** Если X не назван — это не сравнение.

Примеры:
- «Лучше, чем Nivea» → сравнение: X = Nivea ✓
- «Лучше, чем ожидала» → не сравнение (нет X) ✗
- «Как дорогой крем» → спорно (X неточный) → ручной разбор ✓
- «Вместо X пришло Y» → критичное сравнение (подмена) ✓

---

## Шаг 1: Развёртка и выделение X

Для каждого сигнала из primary_signals[сравнение]:
1. Взять unit_index (#)
2. Найти исходный текст
3. Привести **полную цитату**
4. Выделить **жирным** формулировку сравнения
5. Явно указать X — с чем/кем сравнивается товар

Формат:
```
#unit_index | имя | «полная цитата с **выделенным сравнением**» | X = <объект сравнения>
```

Пример:
```
#11 | Таня | «**Лучше, чем крем от Nivea**, и запах приятнее» | X = Nivea
#19 | Лена | «**Запах похож на La Mer**, только цена в 10 раз ниже» | X = La Mer
#28 | Покупатель | «**Вместо заказанного крема пришёл другой товар**» | X = заказанный товар [ПОДМЕНА]
```

---

## Шаг 2: Нормализация X

Объединять X только при **мелких вариациях**:
- Регистр: «Nivea» и «NIVEA» → «Nivea»
- Падеж: «Нивеа» и «Nivea» → «Nivea»
- Ё/е: «Сёёра» и «Сеера» → нормализовать

**Не объединять** по смыслу:
- «аптечный крем» и «крем из аптеки» → можно объединить (одно и то же)
- «Nivea» и «L'Oréal» → разные X, не объединять
- «предыдущий крем» и «старый вариант» → объединить если явно один товар

---

## Шаг 3: Группировка по X

Группировать сигналы по объекту сравнения X.

Для каждого X:
- Перечислить все сравнения
- Указать тональность: лучше / хуже / нейтральное
- Подсчитать R_signal
- Вычислить проценты

Сортировать по убыванию R_signal.

---

## Шаг 4: Критичные сравнения

Выделить отдельным блоком:

**Тип «Подмена»** — «вместо X пришло Y», «не то, что заказывала»:
```
🔴 ПОДМЕНА: N случаев
   #28 | Покупатель | «вместо заказанного крема пришёл другой товар»
```

**Тип «Деградация»** — «раньше было лучше», «предыдущая партия была лучше»:
```
⚠️ ДЕГРАДАЦИЯ: N случаев
   #45 | Ирина | «предыдущая партия была гораздо лучше по запаху»
```

---

## Шаг 5: Подсчёт

```
R_signal = количество уникальных unit_index для данного X
% от B_total = R_signal / B_total × 100
% от B_clean = R_signal / B_clean × 100
```

---

## Шаг 6: Динамика по времени

Если есть даты → отметить появление новых X и изменения.  
Если нет → `NO DATA`

## Жёсткие запреты

- **Не включать сравнения без явного X** — они должны были уйти в ручной разбор на шаге 02
- **Не нормализовать X слишком агрессивно** — объединять только очевидные варианты одного объекта
- **Не интерпретировать тональность** если она неочевидна — указывать «тональность неясна»
