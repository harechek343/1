# Промпт 03: Позитивные сигналы

**Шаг pipeline:** 03  
**Вход:** `primary_signals[]` (только категория «позитив») + исходная таблица + `B_total` + `B_clean`  
**Выход:** `category_report[позитив]`

---

## Задача

Развернуть позитивные сигналы в полные цитаты, сгруппировать в темы, подсчитать частоту, проверить обещания производителя (если предоставлены), выявить паттерны накрутки.

---

## Шаг 1: Развёртка по unit_index

Для каждого сигнала из primary_signals[позитив]:
1. Взять unit_index (#)
2. Найти соответствующую строку в исходной таблице
3. Привести **полную цитату** из оригинального текста
4. Выделить **жирным** конкретный сигнал внутри цитаты

Формат строки:
```
#unit_index | имя | «полная цитата с **выделенным сигналом**»
```

Пример:
```
#1 | Анна | «Крем потрясающий, **запах держится весь день**. Очень довольна.»
#5 | Вера | «Беру уже третий раз. **Замечательный запах, нежный и ненавязчивый.**»
```

---

## Шаг 2: Группировка в темы

Сгруппировать сигналы по предмету позитива. Название темы — **2-4 слова**, почти-цитатная формулировка (не абстрактный тег).

Примеры названий тем:
- «запах лёгкий и держится» ✓
- «аромат» ✗ (слишком абстрактно)
- «быстро впитывается» ✓
- «текстура» ✗ (слишком абстрактно)

Для каждой темы:
- Перечислить все unit_index с цитатами
- Подсчитать R_signal (уникальных отзывов)
- Вычислить R_signal/B_total (%) и R_signal/B_clean (%)

---

## Шаг 3: Подсчёт

Для каждой темы:
```
R_signal = количество уникальных unit_index в теме
% от B_total = R_signal / B_total × 100
% от B_clean = R_signal / B_clean × 100
```

Сортировать темы по убыванию R_signal.

---

## Шаг 4: Блок обещаний производителя

Выполнять только если в начале входных данных предоставлен блок «Обещания производителя».

Для каждого обещания определить:
- **Подтверждается** (есть 2+ конкретных сигнала по теме)
- **Слабое подтверждение** (есть 1 сигнал)
- **Подтвердить не удалось** (нет сигналов или противоречивые данные)

Формат:
```
ОБЕЩАНИЯ ПРОИЗВОДИТЕЛЯ:
✓ Подходит для чувствительной кожи — подтверждается (N=5)
✓ Устойчивый аромат 8+ часов — слабое подтверждение (N=1)
✗ Натуральный состав — подтвердить не удалось (нет упоминаний)
```

---

## Шаг 5: Сигнализация возможной накрутки

Проверить 4 признака:

1. **Однотипность** — ≥30% отзывов используют почти одинаковые фразы
2. **Копипаст** — 2+ отзыва с идентичным или очень похожим текстом
3. **Отсутствие конкретики** — большинство позитивов без конкретных деталей («супер!», «очень понравился»)
4. **Бонусный паттерн** — упоминание подарков или бонусов за отзыв

Вывод:
- 🟢 Признаков накрутки не обнаружено
- 🟡 Слабые признаки накрутки (1 признак): [описание]
- 🔴 Сильные признаки накрутки (2+ признака): [описание]

---

## Шаг 6: Редкие, но сильные позитивы

Выделить отдельным блоком сигналы с R_signal = 1, но высокой информативностью (конкретный эффект, необычный сценарий, сильная эмоция).

---

## Шаг 7: Динамика по времени

Если есть поле «Дата» в исходной таблице:
- Распределить темы по периодам (кварталы или месяцы)
- Отметить, если тема усиливается или ослабевает
- Если нет дат → вывести `NO DATA`

## Жёсткие запреты

- Не включать пустые похвалы без конкретики (они ушли в мусор на шаге 02)
- Не изменять цитаты — только выделять жирным
- Не объединять темы с разными предметами для накрутки счётчика
- Не интерпретировать — только описывать то, что написано
