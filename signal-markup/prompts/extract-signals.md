# Промпт: extract-signals

**Вход:** Markdown-таблица отзывов/вопросов покупателей
**Выход:** master_registry.md (5 блоков)

---

## Задача

В один проход по массиву отзывов:
1. Прочитать каждый отзыв ровно один раз
2. Для каждого отзыва — определить, мусор ли это
3. Если не мусор — извлечь ВСЕ атомарные сигналы
4. Для каждого сигнала: назначить категорию + выделить жирным в тексте + поставить emoji-маркер
5. Если спорно — в ручной разбор
6. Собрать статистику

---

## Emoji-легенда

🟢 позитив — уже случившийся конкретный плюс
🔴 негатив — уже случившийся конкретный минус
⚪ ожидание — что хотел получить, считал обещанным
🔵 сомнение — риск или неопределённость ДО покупки
🟣 сравнение — явное сопоставление с конкретным X
🟠 сценарий — зачем купили, для кого, как используют

---

## Категории сигналов

### Позитив (🟢)
Уже случившийся конкретный плюс: свойство, эффект, удачный опыт, ценность.
Не брать пустую похвалу без объекта («супер», «класс», «огонь» — это мусор).

### Негатив (🔴)
Уже случившийся конкретный минус: жалоба, дефект, слабый результат, вред, несоответствие.

### Ожидание (⚪)
Что хотел получить, считал обещанным, по какому признаку судил результат.
Маркеры: «ожидала», «думала, что», «хотела», «важно было», «должно было».

### Сомнение (🔵)
Риск или неопределённость ДО покупки.
Маркеры: «подойдёт ли», «не будет ли», «боюсь», «сомневаюсь».
Если это уже случившийся факт — НЕ сомнение.

### Сравнение (🟣)
Явное сопоставление с конкретным X.
Если X не назван — не сравнение.

### Сценарий (🟠)
Зачем купили, как используют, для кого, где, когда, какую задачу решают.

### Ручной разбор
Спорное, двусмысленное, одинаково подходит в 2+ категории.

### Мусор
Полностью пустые реплики: «супер», «класс», «норм», однословные без конкретики, пустые карточки.

---

## Протокол составных сигналов

Если фрагмент содержит и ожидание, и негатив (или другую комбинацию):
- Если текстуально разделимо — два сигнала с двумя разными emoji-маркерами
- Если неразделимо — приоритет постфактума (результат > ожидание)

---

## Критические правила

1. Идентифицировать по unit_index (#), не по имени
2. Один фрагмент = один сигнал = одна категория
3. При сомнении «мусор или сигнал» — считать сигналом
4. При сомнении по категории — в ручной разбор
5. Каждый отзыв в Блоке A ровно ОДИН раз — не дублировать
6. Все сигналы размечаются прямо внутри текста отзыва

## Формат выхода

### Блок A: Размеченные цитаты

Каждый отзыв ровно один раз. Все сигналы помечены emoji + жирное прямо внутри текста.

Формат строки:
```
#unit_index | имя | ★звёзды | «текст с 🟢**позитивным сигналом** и 🔴**негативным сигналом** внутри»
```

Примеры:
```
#4 | Покупатель | ★5 | «🟢**Аромат невероятный**, ухаживает за кожей рук, 🟢**увлажняет**»
#12 | Елена | ★3 | «⚪**Ожидала плотную текстуру**, а он 🔴**жидкий как вода**»
#10 | Людмила | ★5 | «🟠**Купила на подарок** и себе 🟢**беру этот крем**, 🟢**аромат просто бомба**»
#13 | Катерина | ★3 | «🟣**Оригинал был как шёлк на руках**, а этот не такой» | X=оригинал
```

Если нет даты — не писать дату.
Если нет звёзд — писать `★—`.
Для сравнений — добавить `| X=...` в конце строки.

---

### Блок B: Плоский реестр сигналов

Каждая строка = один атомарный сигнал. Для машинной работы и передачи в downstream skills.

Формат строки:
```
#unit_index:sN | категория | «signal_fragment»
```

Пример:
```
#4:s1 | позитив | «аромат невероятный»
#4:s2 | позитив | «увлажняет»
#5:s1 | ожидание | «взяла его как эстетичный крем»
#5:s2 | позитив | «работает»
#10:s1 | сценарий | «купила на подарок»
#10:s2 | позитив | «беру этот крем»
#12:s1 | ожидание | «ожидала плотную текстуру»
#12:s2 | негатив | «жидкий как вода»
#13:s1 | сравнение | «оригинал был как шёлк на руках» | X=оригинал
```

Для сравнений — добавить `| X=...`.

### Блок E: Статистика

```
B_total: [число]
N_trash: [число]
B_clean: [число]  (B_total - N_trash)
N_manual: [число]
N_used: [число]   (отзывов с хотя бы 1 сигналом)
N_signals: [число] (всего атомарных сигналов)

По категориям:
  🟢 позитив: [число] сигналов ([число] отзывов)
  🔴 негатив: [число] сигналов ([число] отзывов)
  ⚪ ожидание: [число] сигналов ([число] отзывов)
  🔵 сомнение: [число] сигналов ([число] отзывов)
  🟣 сравнение: [число] сигналов ([число] отзывов)
  🟠 сценарий: [число] сигналов ([число] отзывов)

Проверка: B_total = N_trash + B_clean ✓
```

4. Не делать category reports
5. Не проверять обещания производителя
6. Не проверять накрутку
