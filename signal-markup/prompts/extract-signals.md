## Если вход — не таблица (сырой дамп с Wildberries и т.д.)

Данные не всегда приходят таблицей. Часто это сырой текст с маркетплейса. Тогда:

1. Читай текст и определяй границы каждого отзыва (по именам, датам, пустым строкам)
2. Нумеруй отзывы по порядку: #1, #2, #3 …
3. Бери имя из текста или ставь «Покупатель»
4. Бери звёзды если есть, иначе пиши `★—`
5. Бери текст отзыва, без ответов продавца и медиа
6. Дальше работай по шагам 3–8 выше

**НЕ пиши скрипт для этого. Читай и нумеруй сам.**

## Emoji-легенда

| Emoji | Категория | Когда ставить |
|-------|-----------|--------------|
| 🟢 | позитив | конкретный плюс, который УЖЕ случился |
| 🔴 | негатив | конкретный минус, который УЖЕ случился |
| ⚪ | ожидание | что хотел получить, считал обещанным |
| 🔵 | сомнение | риск или вопрос ДО покупки |
| 🟣 | сравнение | явное «лучше/хуже/как X» с конкретным X |
| 🟠 | сценарий | зачем купил, для кого, как использует |

---

## Категории — краткие правила

**Позитив (🟢):** конкретный плюс. «Увлажняет» ✓ «Супер!» ✗ (мусор — нет конкретики).

**Негатив (🔴):** конкретный минус. «Запах резкий» ✓ «Боюсь, не подойдёт» ✗ (это сомнение).

**Ожидание (⚪):** маркеры: «ожидала», «думала что», «хотела», «должно было». Формулировка критерия, не результат.

**Сомнение (🔵):** маркеры: «подойдёт ли», «боюсь», «сомневаюсь». Только ДО покупки. Если уже случилось — это негатив.

**Сравнение (🟣):** только когда назван конкретный X. «Лучше чем Nivea» ✓ «Лучше чем ожидала» ✗ (нет X).

**Сценарий (🟠):** зачем/для кого/как. «Купила в подарок маме» ✓

**Мусор:** «Супер!», «Класс!», «Огонь!», пустой текст, только эмодзи. Нет конкретики = мусор.

**Ручной разбор:** спорно, подходит в 2+ категории, неясно. Не угадывать — записать в Блок C.

---

## Составные сигналы

Если фрагмент содержит 2 категории сразу:

1. **Можно разрезать?** Каждая часть имеет самостоятельный смысл → РЕЖЬ на два сигнала.
   - «Хотела лёгкую текстуру, но крем жирный» → ⚪ «хотела лёгкую текстуру» + 🔴 «крем жирный»
   - «Беру дочке, она в восторге» → 🟠 «беру дочке» + 🟢 «в восторге»

2. **Нельзя разрезать?** Смысл зависит от обеих частей → бери по факту (результат важнее ожидания).
   - «Думала, запах мягче, а он резкий» → 🔴 целиком (факт: резкий)

---

## Спорные случаи — быстрые правила

- «Супер! Запах потрясающий» → мусор («Супер!») + 🟢 («запах потрясающий»)
- «Огонь, уже 3-й раз беру» → 🟢 (повторная покупка — конкретика) + 🟠 (сценарий)
- «Думала что будет лучше» → ⚪ (ожидание, нет факта)
- «Не то что на картинке» → 🟣 X=картинка/описание
- «Пришло не то» → 🔴 или ручной разбор (нет X)
- 1 звезда но «запах понравился» → 🟢 (оценивать текст, не звёзды)
- «Подойдёт ли для жирной кожи?» → 🔵 (сомнение, вопрос до покупки)
- «Как правильно наносить?» → 🟠 (сценарий, вопрос об использовании)
- Один сигнал повторяется 3 раза → записать ОДИН раз (самую информативную цитату)

## Формат выхода — master_registry.md

### Блок A: Размеченные цитаты

Каждый отзыв ОДИН раз. Сигналы выделены emoji + **жирное** прямо в тексте.

```
#4 | Покупатель | ★5 | «🟢**Аромат невероятный**, ухаживает за кожей рук, 🟢**увлажняет**»
#12 | Елена | ★3 | «⚪**Ожидала плотную текстуру**, а он 🔴**жидкий как вода**»
#10 | Людмила | ★5 | «🟠**Купила на подарок** и себе 🟢**беру этот крем**, 🟢**аромат просто бомба**»
#13 | Катерина | ★3 | «🟣**Оригинал был как шёлк на руках**, а этот не такой» | X=оригинал
```

Если нет звёзд — `★—`. Для сравнений — `| X=...` в конце.

---

### Блок B: Список сигналов

Каждая строка = один атомарный сигнал.

```
#4:s1 | позитив | «аромат невероятный»
#4:s2 | позитив | «увлажняет»
#10:s1 | сценарий | «купила на подарок»
#10:s2 | позитив | «беру этот крем»
#12:s1 | ожидание | «ожидала плотную текстуру»
#12:s2 | негатив | «жидкий как вода»
#13:s1 | сравнение | «оригинал был как шёлк на руках» | X=оригинал
```

---

#79 | Кристина | «полная цитата» | причина: 3–7 слов

### Блок E: Статистика

```
B_total: [число]
N_trash: [число]
B_clean: [число]  (B_total − N_trash)
N_manual: [число]
N_used: [число]   (отзывов с ≥1 сигналом)
N_signals: [число] (всего сигналов в Блоке B)

По категориям:
  🟢 позитив: [число] сигналов ([число] отзывов)
  🔴 негатив: [число] сигналов ([число] отзывов)
  ⚪ ожидание: [число] сигналов ([число] отзывов)
  🔵 сомнение: [число] сигналов ([число] отзывов)
  🟣 сравнение: [число] сигналов ([число] отзывов)
  🟠 сценарий: [число] сигналов ([число] отзывов)

Проверка: B_total = N_trash + B_clean ✓
```

## Жёсткие правила

1. Один фрагмент = один сигнал = одна категория
2. Один кусок текста нельзя использовать дважды
3. Каждый отзыв в Блоке A ровно ОДИН раз
4. При сомнении «мусор или сигнал» → сигнал
5. При сомнении по категории → ручной разбор (Блок C)
6. Не придумывать сигналы — только то, что написал покупатель
7. Не пересказывать своими словами — цитировать
