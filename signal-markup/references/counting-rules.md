# Правила подсчёта: B_total, B_clean, N

Корректный подсчёт — основа сравнимости данных. Все числа выводятся **параллельно** через оба знаменателя.

---

## Основные переменные

### B_total
**Все единицы в входном массиве** — сумма отзывов и вопросов покупателей до любой фильтрации.

```
B_total = количество строк в normalized_units[]
```

Используется для: абсолютных утверждений («63 из 500 упоминают запах»).

---

### N_trash
**Количество мусорных единиц** — полностью пустые реплики, нечитаемый текст, только эмодзи без смысла.

```
N_trash = количество единиц в trash[]
```

---

### B_clean
**Очищенный знаменатель** — после удаления мусора.

```
B_clean = B_total − N_trash
```

Используется для: корректного сравнения (когда позже добавится батч на несколько товаров).

---

### N_signals_raw
**Общее количество сигналов после первичной разметки**, включая ручной разбор.

```
N_signals_raw = N_primary + N_manual
```

---

### N_manual
**Количество единиц в ручном разборе**.

```
N_manual = количество записей в manual_queue[]
```

---

### R_signal
**Сколько отзывов содержат конкретный сигнал/тему** — основная метрика частоты.

```
R_signal = количество уникальных unit_index, в которых встречается данный сигнал
```

---

## Правила дедупликации

### Правило 1: Несколько сигналов из одного отзыва
Если один отзыв содержит 3 разных сигнала — каждый считается **отдельно** для своей категории.

```
Отзыв #12 содержит: позитив "запах приятный" + негатив "флакон неудобный" + сценарий "беру дочке"

→ N[позитив/запах] += 1
→ N[негатив/флакон] += 1
→ N[сценарий/дочка] += 1
```

### Правило 2: Повторяющийся сигнал в одном отзыве
Если один и тот же сигнал упоминается в одном отзыве несколько раз — считается как **1 отзыв**.

```
Отзыв #8: «запах приятный... и запах просто потрясающий... запах — лучшее в этом креме»
→ тема "запах" из отзыва #8: N += 1 (не +3)
```

### Правило 3: N_used vs N_signals
- `N_used` = количество **уникальных отзывов**, из которых извлечён хотя бы один сигнал
- `N_signals` = общее количество **сигналов** (один отзыв может дать несколько сигналов)

```
Если отзывы #3, #7, #12 дали по 3 сигнала каждый:
N_used = 3
N_signals = 9
```

---

## Как считать R_signal для темы

Тема = группа сигналов с одинаковым предметом.

```
Тема "запах": сигналы из отзывов #2, #5, #5, #11, #18
→ Уникальные unit_index: {2, 5, 11, 18}
→ R_signal = 4 (не 5, т.к. отзыв #5 посчитан один раз)
```

---

## Как выводить проценты

**Всегда выводить оба значения параллельно:**

```
Тема "запах": 4 отзыва — 4/B_total (N%) и 4/B_clean (N%)
```

Пример при B_total=120, B_clean=115:
```
Тема "запах": 4 отзыва — 4/120 (3.3%) и 4/115 (3.5%)
```

### Абсолютные утверждения → от B_total
> «63 из 500 покупателей упоминают запах»  
→ 63/B_total

### Корректное сравнение между товарами → от B_clean
→ N/B_clean

---

## Проверочная формула

В финальной статистике обязательно проверять:

```
N_total = N_trash + B_clean
```

Где `N_total` = `B_total`.

Если проверка не сходится — ошибка в подсчёте, нужно найти расхождение.

## Итоговая таблица статистики

| Переменная | Формула | Что означает |
|-----------|---------|-------------|
| B_total | — | Все единицы на входе |
| N_trash | — | Мусор (пустые реплики) |
| B_clean | B_total − N_trash | Информативные единицы |
| N_manual | — | В ручном разборе |
| N_used | — | Уникальных отзывов с сигналами |
| N_signals_raw | — | Сигналов до ручного разбора |
| N_signals_final | — | Сигналов после финализации |
| R_signal | — | Уникальных отзывов с данным сигналом |
