# Формат выходных данных: master_registry.md

Skill `signal-markup` возвращает один документ `master_registry.md` с 5 блоками.

---

## Emoji-легенда

| Emoji | Категория | Описание |
|-------|-----------|---------|
| 🟢 | позитив | Уже случившийся конкретный плюс |
| 🔴 | негатив | Уже случившийся конкретный минус |
| ⚪ | ожидание | Что хотел получить, считал обещанным |
| 🔵 | сомнение | Риск или неопределённость ДО покупки |
| 🟣 | сравнение | Явное сопоставление с конкретным X |
| 🟠 | сценарий | Зачем купили, для кого, как используют |

---

## Блок A: Размеченные цитаты

Каждый отзыв ровно один раз. Все сигналы помечены emoji + **жирное** прямо внутри текста.

### Формат строки

```
#unit_index | имя | ★звёзды | «текст с 🟢**позитивным сигналом** и 🔴**негативным сигналом** внутри»
```

- Если нет звёзд — писать `★—`
- Если нет даты — дату не писать
- Для сравнений — добавить `| X=...` в конце строки

### Примеры

```
#4 | Покупатель | ★5 | «🟢**Аромат невероятный**, ухаживает за кожей рук, 🟢**увлажняет**»
#12 | Елена | ★3 | «⚪**Ожидала плотную текстуру**, а он 🔴**жидкий как вода**»
#10 | Людмила | ★5 | «🟠**Купила на подарок** и себе 🟢**беру этот крем**, 🟢**аромат просто бомба**»
#13 | Катерина | ★3 | «🟣**Оригинал был как шёлк на руках**, а этот не такой» | X=оригинал
```

---

## Блок B: Плоский реестр сигналов

Каждая строка = один атомарный сигнал. Для машинной работы и передачи в downstream skills.

### Формат строки

```
#unit_index:sN | категория | «signal_fragment»
```

- `sN` — порядковый номер сигнала внутри одного отзыва (s1, s2, s3 …)
- Для сравнений — добавить `| X=...`

### Примеры

```
#4:s1 | позитив | «аромат невероятный»
#4:s2 | позитив | «увлажняет»
#5:s1 | ожидание | «взяла его как эстетичный крем»
#5:s2 | позитив | «работает»
#10:s1 | сценарий | «купила на подарок»
#10:s2 | позитив | «беру этот крем»
#12:s1 | ожидание | «ожидала плотную текстуру»
#12:s2 | негатив | «жидкий как вода»
#13:s1 | сравнение | «оригинал был как шёлк на руках» | X=оригинал
```

---

## Блок C: Ручной разбор

Спорные и двусмысленные отзывы, которые не удалось классифицировать уверенно.

### Формат строки

```
#unit_index | имя | «полная цитата целиком» | причина: 3–7 слов
```

### Пример

```
#79 | Кристина | «полная цитата целиком» | причина: категория неочевидна
#163 | Ирина | «полная цитата целиком» | причина: до или после покупки?
```

---

## Блок D: Мусор

Полностью пустые реплики и тексты без конкретики.

### Формат строки

```
#unit_index | имя | «оригинальный текст или [пусто]»
```

### Пример

```
#57 | Оксана | «[пусто]»
#134 | Виктория | «Это космос 🤪❤️»
#22 | Покупатель | «Супер!!!»
```

---

## Блок E: Статистика

### Формат

```
B_total: [число]
N_trash: [число]
B_clean: [число]  (B_total - N_trash)
N_manual: [число]
N_used: [число]   (отзывов с хотя бы 1 сигналом)
N_signals: [число] (всего атомарных сигналов)

По категориям:
  🟢 позитив: [число] сигналов ([число] отзывов)
  🔴 негатив: [число] сигналов ([число] отзывов)
  ⚪ ожидание: [число] сигналов ([число] отзывов)
  🔵 сомнение: [число] сигналов ([число] отзывов)
  🟣 сравнение: [число] сигналов ([число] отзывов)
  🟠 сценарий: [число] сигналов ([число] отзывов)

Проверка: B_total = N_trash + B_clean ✓
```

### Описание полей

| Поле | Описание |
|------|---------|
| `B_total` | Всего отзывов на входе |
| `N_trash` | Отзывов, отправленных в мусор |
| `B_clean` | Отзывов с потенциальным содержанием (B_total − N_trash) |
| `N_manual` | Отзывов, отправленных в ручной разбор |
| `N_used` | Отзывов, из которых извлечён хотя бы 1 сигнал |
| `N_signals` | Всего атомарных сигналов в Блоке B |
