# Типичные спорные случаи (edge cases)

Реальные ситуации, которые возникают при разметке. Для каждого случая — правило решения.

---

## 1. «Покупатель» без уникального имени

**Ситуация:** несколько единиц с именем «Покупатель», «Аноним», «Пользователь» или без имени.

**Решение:**
- Каждая единица идентифицируется по `unit_index` (#), не по имени
- Имя «Покупатель» сохраняется как есть — не изменять, не нумеровать
- Если два «Покупателя» с одним unit_index — это ошибка входных данных; отправить в ручной разбор с причиной «дублирование unit_index»

---

## 2. Несколько отзывов от одного имени

**Ситуация:** «Анна» встречается 5 раз с разными отзывами.

**Решение:**
- Каждый отзыв — самостоятельная единица с уникальным unit_index
- Имя Анна — просто видимый идентификатор, не уникальный ключ
- Для подсчёта R_signal — каждый unit_index считается отдельно

---

## 3. Пустая похвала: «супер», «класс», «огонь», «5/5»

**Ситуация:** текст отзыва содержит только восклицание без конкретики.

**Правило:** при сомнении «мусор или сигнал» — считать сигналом. **Но** «супер», «класс», «огонь» — это **пустая похвала без конкретики**.

**Решение:**
- «Супер!» — мусор (`trash[]`), не позитив
- «Огонь, советую всем» — мусор
- «5 звёзд» без текста — мусор
- «Класс! Запах потрясающий» — мусор + позитив (разрезать: «Класс!» в мусор, «запах потрясающий» в позитив если есть конкретика)
- «Огонь, уже 3-й раз беру» — позитив (конкретика: повторная покупка) + сценарий

---

## 4. Фраза одновременно подходит в 2 категории

**Ситуация:** «Думала, что будет лучше» — это ожидание или негатив?

**Решение:**
1. Попробовать разрезать: есть ли явный факт И явное ожидание?
2. Если неразделимо → применить [`composite-signal-protocol.md`](composite-signal-protocol.md) (приоритет постфактума)
3. Если даже с протоколом неясно → ручной разбор

**Конкретный случай:**
- «Думала, что будет лучше» — нет факта, только ожидание → `ожидание`
- «Думала, что будет лучше, но разочарована» — есть ожидание + факт разочарования → протокол составных сигналов
- «Думала, что будет лучше» в контексте 1 звезды — всё равно ожидание без явного негатива → `ожидание`, не угадывать

---

## 5. Отзыв на 1 звезду с позитивным фрагментом

**Ситуация:** покупатель поставил 1 звезду, но в тексте есть «запах понравился».

**Правило:** оценивать **текст**, не звёзды. Звёздная оценка — метаданные, не сигнал.

**Решение:**
- «1 звезда, запах понравился, но флакон треснул» → позитив («запах понравился») + негатив («флакон треснул»)
- Звезду не использовать как сигнал — только как метаданные для динамики
- Не игнорировать позитивные фрагменты в низкооценочных отзывах

---

## 6. Вопрос без ожидания и без сомнения

**Ситуация:** «Как правильно наносить?» — это явно вопрос, но ни ожидания, ни сомнения нет.

**Решение:**
- Если вопрос содержит неопределённость до покупки → сомнение
- Если вопрос об использовании без признаков сомнения → сценарий (`[вопрос об использовании]`)
- Если вопрос нейтральный и не классифицируется → ручной разбор

**Конкретно:**
- «Как правильно наносить?» → сценарий: [вопрос об использовании]
- «Подойдёт ли для жирной кожи?» → сомнение: [вопрос без ответа]
- «Когда ждать результата?» → сомнение или ожидание (зависит от контекста) → ручной разбор если неясно

---

## 7. «Не то» — сравнение или негатив?

**Ситуация:** «Не то, что я ожидала» или «Пришло не то».

**Правило:** сравнение требует явного X. Если X не назван — не сравнение.

**Решение:**
- «Не то, что я ожидала» — ожидание (есть «ожидала») + негатив (есть несоответствие) → протокол составных сигналов
- «Пришло не то» — нет X → негатив (несоответствие заказу) ИЛИ ручной разбор
- «Не то, что на картинке» — сравнение: X = картинка/описание
- «Пришло не то, что заказывала» — негатив: тип «несоответствие заказу», не сравнение

---

## 8. «Как X» без явного X

**Ситуация:** «Запах как что-то французское», «Как дорогой крем».

**Правило:** сравнение требует конкретного X. «Что-то французское» — не конкретный X.

**Решение:**
- «Как что-то французское» → не сравнение → позитив (если оценка положительная) или ручной разбор
- «Как дорогой крем» → сравнение спорное: X неточный, но есть смысл → ручной разбор с причиной «X неточный»
- «Как La Mer» → сравнение: X = La Mer ✓

---

## 9. Одинаковый сигнал в одном отзыве несколько раз

**Ситуация:** «Запах отличный, правда отличный, просто потрясающий запах» — одна тема трижды.

**Решение:**
- Одна тема из одного отзыва = 1 в R_signal
- В primary_signals фиксируем один сигнал (самую информативную цитату)
- Дублирующие упоминания не добавляют в N

---

## 10. Отзыв-вопрос смешанного типа

**Ситуация:** «Крем хороший, но подойдёт ли он после 40?» — смесь позитива и сомнения.

**Решение:**
- Разрезать: «крем хороший» → позитив (хотя и без конкретики — ручной разбор)
- «подойдёт ли он после 40?» → сомнение: [вопрос без ответа]
- Оба сигнала от одной единицы с одним unit_index
